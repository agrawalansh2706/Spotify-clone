# Phase 5: Spotify Web API Authentication - Execution Plan

## 05-01: NextAuth Configuration & Identity

### Intent
Configure NextAuth to handle Spotify login and securely manage Spotify access tokens.

### Dependencies
- Install `next-auth` (v4)
- User must supply `SPOTIFY_CLIENT_ID` and `SPOTIFY_CLIENT_SECRET` via `.env.local`

### Steps
1. **Dependencies**: `npm install next-auth`
2. **NextAuth API Route**: Create `src/app/api/auth/[...nextauth]/route.ts`. Configure the `SpotifyProvider` with scopes requested: `streaming user-read-email user-read-private user-library-read user-library-modify user-read-playback-state user-modify-playback-state user-read-recently-played playlist-read-private`.
3. **Token Rotation**: Implement custom `jwt` and `session` callbacks in the NextAuth configuration to ensure the `access_token` and `refresh_token` are saved and the access token is returned down to the client so the player can use it.
4. **Auth Provider Component**: Create `src/components/AuthProvider.tsx` mapping to `<SessionProvider>` to wrap the app layout.
5. **Update Root Layout**: Apply `<AuthProvider>` to wrap the node tree inside `layout.tsx`.
6. **Sidebar Integration**: Update `Sidebar.tsx` to read `useSession()`. Show an avatar/name block at the bottom, or a "Log in" button wrapping `signIn('spotify')` if no session is active.
7. **Environment Stub**: Create `.env.example` as a template for keys.

### Expected Outcome
The app layout correctly fetches session state in the browser. Clicking login in the Sidebar successfully bridges through the Spotify Auth gateway, returns, and displays the user's name/picture.
