# Phase 6: Fetch User Library & Playlists - Execution Plan

## 06-01: API Hooks and Data Fetching

### Intent
Create a utility function to connect to the Spotify Web API with the NextAuth session token, and update the Sidebar and Home to use real user data.

### Dependencies
- NextAuth session must be accessible.

### Steps
1. **API Utility**: Create `src/lib/spotify.ts`. Export a function `customFetch(endpoint, session)` that prepends `https://api.spotify.com/v1` and injects the Bearer token.
2. **Data Hooks**: Create `src/hooks/useSpotifyData.ts` using `swr` or raw `useEffect` to fetch:
   - `/me/playlists` (Sidebar)
   - `/me/top/tracks` (Home View)
3. **Sidebar Re-wire**: Update `Sidebar.tsx` to call the `useSpotifyData` hook. If `session` exists, use the real playlists. If pending or no session, use fallback/mock data or empty state.
4. **Home Page Re-wire**: Update `page.tsx` to display real Spotify Top Tracks or Playlists instead of mock hardcoded grid.
5. **Playlist Detail Page Re-wire**: Update `app/playlist/[id]/page.tsx` to handle the dynamic `id` passed from the URL to fetch the specific playlist data with `/playlists/[id]`. Extract real tracks.

### Expected Outcome
When a user logs in, the Sidebar magically populates with their actual Spotify playlists. Clicking them navigates to `/playlist/[id]` which successfully fetches and lists real Spotify tracks with accurate names, artists, duration, and album art.
