# Phase 7: Spotify Web Playback SDK - Execution Plan

## 07-01: Update Global Store and Integrate SDK

### Intent
Integrate the Spotify Web Playback SDK, mount the player when authenticated, and rewire play/pause controls to use the real Spotify Player or Spotify Connect API.

### Dependencies
- `@types/spotify-web-playback-sdk` (for TS support)
- NextAuth session access token.

### Steps
1. **Types**: Add `npm i -D @types/spotify-web-playback-sdk` so we get proper types for `Spotify.Player`.
2. **Setup Store**: Modify `src/store/playerStore.ts`. Add `deviceId`, `player`, `isReady`, and `playbackError` to the store. Remove HTML `audioRef`. 
   - Add actions: `setPlayer`, `setDeviceId`, `syncState` (to update UI from SDK events).
3. **Player Initializer Component**: Create `src/components/SpotifyPlayer.tsx`.
   - Use Next.js `<Script>` to load `https://sdk.scdn.co/spotify-player.js`.
   - On load, initialize `new window.Spotify.Player(...)` passing the NextAuth session token.
   - Attach listeners for `ready`, `not_ready`, `player_state_changed`, and error events.
   - Update `playerStore` with state changes.
4. **Update Layout**: Wrap `SpotifyPlayer` in `src/app/layout.tsx` (inside `AuthProvider`).
5. **Update PlayerBar and TrackRow**: 
   - In `PlayerBar` and `TrackRow`, change the "play" buttons to either use `player.togglePlay()` (if currently active) or perform a `PUT /me/player/play?device_id=X` API call to begin streaming a specific URI.
   - Note: Spotify SDK requires URIs in the format `spotify:track:ID`.

### Expected Outcome
When a Premium user logs in, a "Spotify Clone Web Player" device becomes available. Playing a track streams real music securely via encrypted pipelines provided by the officially supported Spotify script, updating our UI accordingly.
